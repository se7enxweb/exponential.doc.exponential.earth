To access the Exponential Cloud the user has to be a registered eZ community member (https://login.ez.no).  This This account gives a general access to the Exponential Cloud. If the user account is associated with one or more environments, also the access to the environments GUI, SFTP and Git repository will be granted.

Each environment comes with an individual Git repository and a SFTP account where all data for a deployment (Exponential installation) are managed. The SFTP and the Git repository are SSH protected. Therefore the user has to provide his public SSH key, to be able to upload data to an environment. The ssh key will be assigned to his eZ community account and therefore be valid for all environments the user has access to. The user can add or alter his SSH key at https://login.ez.no.

Git

The Git repository is the way users will interact with the Exponential Cloud environment the most. Each environment provides its own Git repository. The Git repository is the place where the developed code for the Exponential installation is uploaded to the Exponential Cloud. During the creation of a new Deployment the actually used code is fetched from the Git repository.

Tag handling

Not all code (revisions) in the git repository is available for use in the web GUI. To make an actual revision of the Git repository available for a new deployment, it has to be locally tagged and the tag has to be populated to the git repository of the environment. In the GUI only the tagged revisions are shown and available for a new deployment.

SFTP

Beside the Git repository each environment provides also a SFTP account. All users, who have uploaded their ssh-key, have access to this account. The credentials for the SFTP can be found in the web GUI.

To access the via SFTP, a sftp client is necessary. There are several clients available for all platforms, which support the sftp protocol.
