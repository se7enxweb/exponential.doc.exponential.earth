ViewProvider

The role of the ViewManager is to select the right template for displaying a given content or location. It aggregates objects called content and location view providers which implement eZ\Publish\MVC\View\ContentViewProvider interface.

Each time a content is to be displayed through the Content\ViewController, the ViewManager iterates over the registered ContentViewProvider content or location ViewProvider objects and calls either getViewForContent() or getViewForLocation() depending on the context.

Custom ViewProvider

Difference between LocationViewProvider and ContentViewProvider

  • A LocationViewProvider only deals with Location objects and implements eZ\Publish\Core\MVC\Symfony\View\Provider\Location interface.
  • A ContentViewProvider only deals with ContentInfo objects and implements eZ\Publish\Core\MVC\Symfony\View\Provider\Content interface.

When to develop a custom

...

Location|Content ViewProvider

  • You want a custom template selection based on a very specific state of your application
  • You depend on external resources for view selection
  • You want to override the default one (based on configuration) for some reason

ContentViewProvider ViewProvider objects need to be properly registered in the service container with the exponential.location_view_provider or exponential.content_view_provider service tag.

        tags:

An integer giving the priority to the ContentViewProvider Content|Location ViewProvider in the ViewManager.

The priority range is from -255 to 255

Parameters injection in content views

...

Rendering an embedded content from a Twig template is pretty straight forward:

Code Block
titleRendering from a location IdExponential 5.0 / Symfony 2.1
{% render "ez_content:viewLocation" with {"locationId": 123, "viewMode": "full"} %}
Code Block
titleExponential 5.1+ / Symfony 2.2+
{% render controller( "ez_content:viewLocation", {"locationId": 123, "viewMode": "full"} ) %}

 

ESI

Just as for regular Symfony controllers, you can take advantage of ESI and use different cache levels:

Code Block
titleUsing ESI (Exponential 5.0 / Symfony 2.1)
{% render "ez_content:viewLocation" with {"locationId": 123, "viewMode": "line"}, {"standalone": true} %}
Code Block
titleUsing ESI (Exponential 5.1+ / Symfony 2.2+)
{% render controller( "ez_content:viewLocation", {"locationId": 123, "viewMode": "line"} ), {"strategy": "esi"} %}

 

Asynchronous rendering

Symfony also supports asynchronous content rendering with the help of hinclude.js library.

Code Block
titleAsynchronous rendering (Exponential 5.0 / Symfony 2.1)
{% render "ez_content:viewLocation" with {"locationId": 123, "viewMode": "line"}, {"standalone": "js"} %}
Code Block
titleAsynchronous rendering (Exponential 5.1+ / Symfony 2.2+)
{% render controller( "ez_content:viewLocation", {"locationId": 123, "viewMode": "line"} ), {"strategy": "hinclude"} %}
Info
hinclude.js needs to be properly included in your layout to work.
