View\Manager & View\Provider

The role of the ViewManager is (eZ\Publish\Core\MVC\Symfony\)View\Manager is to select the right template for displaying a given content or location. It aggregates objects called content and location view providers that  which respectively implement eZ\Publish\Core\MVC\View\ContentViewProvider interfaceSymfony\View\Provider\Content and eZ\Publish\Core\MVC\Symfony\View\Provider\Location interfaces.

Each time a content is to be displayed through the Content\ViewController, the ViewManager View\Manager iterates over the registered ContentViewProvider content or location View\Provider objects and calls either getViewForContentcalls getView() or getViewForLocation() depending on the context.

Custom View\Provider

Difference between View\Provider\Location and View\Provider\Content

  • A View\Provider\Location only deals with Location objects and implements eZ\Publish\Core\MVC\Symfony\View\Provider\Location interface.
  • A View\Provider\Content only deals with ContentInfo objects and implements eZ\Publish\Core\MVC\Symfony\View\Provider\Content interface.

When to develop a custom

...

View\Provider\(Location|Content)

  • You want a custom template selection based on a very specific state of your application
  • You depend on external resources for view selection
  • You want to override the default one (based on configuration) for some reason

ContentViewProvider View\Provider objects need to be properly registered in the service container with the exponential.location_view_provider or exponential.content_view_provider service tag.

Render embedded content objects

Rendering an embedded content from a Twig template is pretty straight forward:

Code Block
titleRendering from a location Id
{% render "ez_content:viewLocation" with {"locationId": 123, "viewMode": "full"} %}

ESI

Just as for regular Symfony controllers, you can take advantage of ESI and use different cache levels:

Code Block
titleUsing ESI
{% render "ez_content:viewLocation" with {"locationId": 123, "viewMode": "line"}, {"standalone": true} %}

Asynchronous rendering

Symfony also supports asynchronous content rendering with the help of hinclude.js library.
